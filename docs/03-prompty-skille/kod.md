# Prompty – Kod i development

## Code Review

```
Zrób szczegółowy code review poniższego kodu.
Język: [PHP/Python/JS/inne]
Kontekst: [co ten kod robi]

[WKLEJ KOD]

Oceń:
1. Poprawność logiczna
2. Bezpieczeństwo (SQL injection, XSS, itp.)
3. Wydajność
4. Czytelność i standardy kodowania
5. Możliwe edge cases i błędy

Podaj konkretne sugestie poprawek z fragmentami kodu.
Priorytetyzuj: KRYTYCZNE / WAŻNE / SUGESTIE
```

## Debugowanie

```
Mam błąd w kodzie. Pomóż mi go zdebugować.

Język/framework: [np. PHP 8.2 / WordPress]
Błąd (komunikat): [WKLEJ BŁĄD]
Oczekiwane działanie: [co powinno się stać]
Rzeczywiste działanie: [co się dzieje]

Kod:
[WKLEJ KOD]

1. Zdiagnozuj przyczynę błędu
2. Zaproponuj fix z wyjaśnieniem
3. Zasugeruj jak unikać tego typu błędów w przyszłości
```

## Pisanie funkcji/klasy

```
Napisz [funkcję/klasę/moduł] w [JĘZYKU].

Zadanie: [OPIS CO MA ROBIĆ]
Input: [OPIS DANYCH WEJŚCIOWYCH]
Output: [OCZEKIWANY WYNIK]
Wymagania: [dodatkowe wymagania np. "bez zewnętrznych bibliotek"]

Standardy:
- Komentarze w kodzie (po polsku)
- Obsługa błędów / edge cases
- Przykład użycia na końcu
```

## Refaktoryzacja kodu

```
Zrefaktoryzuj poniższy kod zachowując jego funkcjonalność.

Cel refaktoryzacji: [np. "lepsza czytelność", 
"wydajność", "SOLID principles", "DRY principle"]

[WKLEJ KOD]

1. Podaj zrefaktoryzowaną wersję
2. Wyjaśnij każdą znaczącą zmianę
3. Oceń czy zmiana wydajności jest istotna
```

## Dokumentacja / README

```
Napisz dokumentację dla poniższego kodu/projektu.

[WKLEJ KOD lub OPIS PROJEKTU]

Dokumentacja powinna zawierać:
- Opis czym jest projekt (2-3 zdania)
- Wymagania (dependencies, wersje)
- Instalacja krok po kroku
- Konfiguracja (zmienne środowiskowe, config)
- Użycie z przykładami
- API Reference (jeśli dotyczy)
- FAQ / Troubleshooting

Format: Markdown (README.md)
```
